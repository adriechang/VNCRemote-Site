VNC Remote Helper 安裝教學

適用於 macOS | 版本 1.0
什麼是 Helper？
VNC Remote Helper 是一個選用的 macOS 輔助工具。安裝後，您可以在遠端桌面中使用注音、拼音、日文等非英文輸入法。不安裝 Helper 也能正常使用 VNC Remote，只是無法使用輸入法。

為什麼需要 Helper？

VNC 協定只能傳送英文按鍵訊號，無法觸發 macOS 的輸入法系統（如注音、拼音）。Helper 透過在 macOS 端直接注入鍵盤事件，讓輸入法能正常運作。

運作架構：

  • VNC 通道（port 5900）：傳輸螢幕畫面和滑鼠操作
  • Helper 通道（port 50001）：傳輸鍵盤事件，讓輸入法可以辨識

安裝步驟

1 下載原始碼

打開 Mac 的「終端機」(Terminal)，執行：

git clone https://github.com/adriechang/VNCRemote-Site.git
cd VNCRemote-Site
注意：目前 Helper 尚未提供預編譯版本，需要從原始碼編譯。您的 Mac 需要安裝 Xcode Command Line Tools。如果還沒有安裝，終端機會自動提示您安裝。

2 編譯 Helper

cd HIDInjectorHelper
make build

編譯成功後會產生 HIDInjectorHelper 執行檔。

3 執行 Helper

./HIDInjectorHelper

成功啟動後會看到類似訊息：

🚀 Starting HID Injector Helper v1.0
✅ HID Injector Server listening on port 50001
🔑 PIN: 123456

請記下顯示的 PIN 碼，iPad 連線時需要輸入。

4 授予 Accessibility 權限

macOS 會提示需要「輔助使用」權限：

  1. 打開「系統設定」→「隱私權與安全性」→「輔助使用」
  2. 找到 HIDInjectorHelper（或 Terminal）
  3. 啟用權限
  4. 重新啟動 Helper（授權後需要重啟才會生效）

使用方式

在 Mac 上

  1. 確保「系統設定」→「共享」→「螢幕共享」已啟用
  2. 執行 Helper：./HIDInjectorHelper
  3. 確認看到「listening on port 50001」

在 iPad 上

  1. 連接外接鍵盤（藍牙或有線）
  2. 開啟 VNC Remote App → 連線到 Mac
  3. 連線成功後，App 會自動嘗試連接 Helper
  4. 輸入 Mac 上顯示的 PIN 碼

測試輸入法

  1. 在遠端 Mac 上打開任一文字編輯器（TextEdit、備忘錄等）
  2. 按 Control + Space 切換到注音（或其他輸入法）
  3. 用 iPad 外接鍵盤輸入，例如：ㄉㄚˋ
  4. 應該會看到選字候選清單

常見問題

Helper 啟動失敗

可能是 port 50001 被佔用。檢查方法：

lsof -i :50001

如果有其他程式佔用，先關閉該程式再重試。

iPad 無法連接 Helper

  • 確認 iPad 和 Mac 在同一個區域網路
  • 檢查 macOS 防火牆是否擋住了 port 50001
  • 確認 PIN 碼輸入正確

連上了但輸入法不工作

  • 確認已授予 Accessibility 權限
  • 授權後需要重新啟動 Helper
  • 確認 Mac 上已安裝並切換到目標輸入法

不安裝 Helper 可以嗎？

可以。不安裝 Helper 的話，VNC Remote 的所有功能都正常運作，只是無法使用中文、日文等輸入法。英文輸入和所有快捷鍵不受影響。

VNC Remote Helper Setup Guide

For macOS | Version 1.0
What is Helper?
VNC Remote Helper is an optional macOS companion tool. Once installed, you can use Chinese (Zhuyin/Pinyin), Japanese, Korean, and other non-English input methods on the remote desktop. VNC Remote works fine without Helper — you just won't be able to use input methods.

Why Do I Need Helper?

The VNC protocol can only send English key signals and cannot trigger macOS input method systems (e.g., Zhuyin, Pinyin). Helper works by injecting keyboard events directly on the macOS side, allowing input methods to function properly.

Architecture:

  • VNC Channel (port 5900): Transmits screen display and mouse operations
  • Helper Channel (port 50001): Transmits keyboard events for input method recognition

Installation

1 Download Source Code

Open Terminal on your Mac and run:

git clone https://github.com/adriechang/VNCRemote-Site.git
cd VNCRemote-Site
Note: Pre-built binaries are not yet available. You'll need to compile from source. Your Mac needs Xcode Command Line Tools installed. If not already installed, Terminal will prompt you automatically.

2 Build Helper

cd HIDInjectorHelper
make build

On success, the HIDInjectorHelper executable will be created.

3 Run Helper

./HIDInjectorHelper

On successful launch, you'll see:

🚀 Starting HID Injector Helper v1.0
✅ HID Injector Server listening on port 50001
🔑 PIN: 123456

Note the PIN code displayed — you'll need it when connecting from iPad.

4 Grant Accessibility Permission

macOS will prompt for Accessibility access:

  1. Open System Settings → Privacy & Security → Accessibility
  2. Find HIDInjectorHelper (or Terminal)
  3. Enable the toggle
  4. Restart Helper (required after granting permission)

Usage

On Your Mac

  1. Ensure Screen Sharing is enabled (System Settings → Sharing)
  2. Run Helper: ./HIDInjectorHelper
  3. Confirm you see "listening on port 50001"

On Your iPad

  1. Connect an external keyboard (Bluetooth or wired)
  2. Open VNC Remote App → Connect to your Mac
  3. After connecting, the app will automatically attempt to connect to Helper
  4. Enter the PIN code shown on your Mac

Test Input Methods

  1. Open any text editor on the remote Mac (TextEdit, Notes, etc.)
  2. Press Control + Space to switch input methods
  3. Type using your iPad external keyboard
  4. You should see the input method candidate list appear

Troubleshooting

Helper Fails to Start

Port 50001 may be in use. Check with:

lsof -i :50001

If another process is using the port, close it and try again.

iPad Can't Connect to Helper

  • Ensure iPad and Mac are on the same local network
  • Check if macOS firewall is blocking port 50001
  • Verify the PIN code is entered correctly

Connected but Input Methods Don't Work

  • Confirm Accessibility permission has been granted
  • Restart Helper after granting permission
  • Make sure the target input method is installed and selected on Mac

Do I Need Helper?

No. Without Helper, all VNC Remote features work normally — you just can't use Chinese, Japanese, or other non-English input methods. English input and all keyboard shortcuts work fine without it.
