Frequently Asked Questions

Why can't other remote desktop apps handle CJK input methods?

Traditional VNC protocol sends X11 keysym codes, which only represent individual characters and cannot trigger macOS input method composition (e.g., Zhuyin phonetic sequences, Pinyin romanization). VNC Pad solves this by running a companion Helper app on macOS that receives HID usage codes over a separate TCP channel (port 50001) and injects them at the CGEvent HID tap level. macOS treats these as physical keyboard events, enabling native input method processing.

Can I use VNC Pad without the Helper app?

Yes. Without Helper, all VNC Pad features work normally — you just can't use Chinese, Japanese, Korean, or other non-English input methods. English input and all keyboard shortcuts work fine without it.

Can I connect over the internet?

VNC Pad is designed for local network use. For internet access, we recommend using a VPN to place both devices on the same virtual network.
